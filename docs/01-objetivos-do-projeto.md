# Objetivos do Projeto

## Objetivo Principal

Criar a documentação técnica e o mapeamento analítico completo das telas atuais do Power BI (EBITDA, Rentabilidade e Custos) para subsidiar a refatoração e posterior desenvolvimento de uma interface web dinâmica em HTML, garantindo fidelidade de dados, regras de negócio consistentes e uma experiência visual de alto nível.

## Objetivos Secundários

- **Mapeamento de DAX**: Levantar e documentar todas as medidas DAX de cada painel.
- **Identificação de Regras de Negócio**: Explicitá-las em formato texto para que possam ser validadas pela equipe de negócios.
- **Padronização Visual**: Estabelecer um Design System inicial (paleta de cores, tipografia, estrutura de cards de KPI e modelos de gráficos).
- **Integração com API**: Planejar os fluxos de automação de metadados via API do Power BI para acelerar a extração técnica.
- **Definição de Prompts**: Consolidar a atuação da equipe de agentes autônomos Antigravity para as próximas fases.

## Ganhos Esperados

- **Independência Tecnológica**: Possibilidade de hospedar painéis analíticos diretamente em portais corporativos sem a dependência exclusiva de licenças premium de visualização do Power BI para todos os usuários.
- **Usabilidade Executiva**: Redução do tempo de carregamento e melhoria na usabilidade móvel.
- **Documentação Viva**: Criação de um repositório centralizado que serve como dicionário de dados e regras de negócio de BI da empresa.

## Critérios de Sucesso

- Mapeamento de 100% das medidas DAX necessárias para as três telas.
- Aprovação da proposta de layout visual (UX/UI) pelas partes interessadas.
- Estruturação de componentes HTML reutilizáveis especificada.
- Validação técnica de que os dados consumidos pela futura aplicação em HTML corresponderão exatamente aos dados apresentados no Power BI original.

## Resultado Esperado ao Final

Um repositório contendo toda a documentação das telas antigas mapeadas, a proposta de arquitetura de dados e de layout da nova solução, os prompts configurados para a automação de agentes de IA e o backlog técnico pronto para a etapa de desenvolvimento em código HTML.
